# FMS 部署教程

本项目后端由三个服务组成（共用同一个 MySQL 数据库）：

| 端口 | 服务 | 职责 |
|---|---|---|
| 17306 | `auth_api` | 认证 / 注册 / Turnstile 验证接口（**仅内部**，不再对外暴露端口） |
| 1145 | `admin_panel` | 管理后台 UI（用户、SMTP、Turnstile、日志） |
| 3090 | `register_ui` | 用户注册页面，并反向代理 `/api/*` 到 `auth_api` |

> v1.0.5 起 `auth_api` 不再向宿主机暴露端口。浏览器对 `auth_api` 的所有请求统一通过 `register_ui:3090` 的同源反向代理转发，因此不再需要配置 `APP_REGISTER_PUBLIC_AUTH_URL`，也不存在跨域问题。

---

## 1. 需要准备的东西

1. 一台 Linux 服务器（已装 Docker + Docker Compose）
2. 域名解析到服务器公网 IP（可选，但推荐）
3. 腾讯企业邮 SMTP 账号
4. Cloudflare Turnstile 的 `site key` 和 `secret key`

需要对公网开放的端口：

- `3090/tcp`（用户注册页）
- `1145/tcp`（管理后台）
- `80/443`（如使用 Nginx + HTTPS 反代）

> `17306` 不再需要对公网开放。

---

## 2. 目录结构

仓库内：

- `deploy/docker-compose.yml`：统一编排
- `deploy/backup_auth/`：认证 API
- `deploy/admin_panel/`：管理后台
- `deploy/register_ui/`：用户注册页（含反向代理）

建议把整个 `deploy/` 目录上传到 `/opt/fms/deploy`。

---

## 3. 首次部署

```bash
cd /opt/fms/deploy
cp .env.example .env
chmod 600 .env
# 必改：MYSQL_PASSWORD / MYSQL_ROOT_PASSWORD / APP_JWT_SECRET / APP_ADMIN_PASSWORD
docker compose up -d --build
```

`APP_REGISTER_PUBLIC_AUTH_URL` 留空即可（前端会走同源 `/api/*`）。

检查：

```bash
docker compose ps
curl http://127.0.0.1:3090/healthz
curl http://127.0.0.1:1145/healthz
```

---

## 4. 在管理后台完成 SMTP / Turnstile 配置

SMTP 与 Turnstile 都存放在数据库 `app_settings` 表，统一在管理后台填写：

1. 浏览器打开 `http://<server>:1145/`
2. 使用 `APP_ADMIN_USERNAME` / `APP_ADMIN_PASSWORD` 登录
3. **SMTP 设置**：填主机/用户名/密码/发件人，保存后用"发送测试邮件"验证
4. **Turnstile 设置**：填 `site_key` / `secret_key`（留空则跳过人机校验）

> 未配置 SMTP 时，注册接口请求验证码会返回 503。

腾讯企业邮常用配置：

- `host = smtp.exmail.qq.com`
- `port = 465`，`use_ssl = true`
- 用户名 = **完整邮箱地址**（不是显示名）
- 密码 = 企业邮箱后台生成的"**客户端专用密码**"

---

## 5. Cloudflare Turnstile 域名白名单

在 Cloudflare 后台 **Turnstile → 你的 Site → Settings → Hostname Management**，把所有访问注册页的域名加进去：

- 正式域名（如 `register.example.com`）
- 本地测试时还需加 `localhost`、`127.0.0.1`

否则浏览器加载 Turnstile widget 会出现 `110200` 错误（site_key 与 hostname 不匹配）。

测试期间可使用 Cloudflare 提供的"始终通过"测试密钥：

| site_key | secret_key |
|---|---|
| `1x00000000000000000000AA` | `1x0000000000000000000000000000000AA` |

---

## 6. 注册流程

注册页（`http://<server>:3090/`）提交时：

1. 浏览器加载 Turnstile widget
2. 用户填姓名 / 邮箱 / 密码后点击"发送验证码"
3. 前端 POST `/api/auth/register/code` → `register_ui` 反代到 `auth_api`
4. `auth_api` 校验 Turnstile token → 发送 SMTP 邮件
5. 用户输入收到的验证码并提交注册

---

## 7. 升级 / 重置

```bash
cd /opt/fms/deploy
git pull
docker compose up -d --build
```

彻底清空数据库（会丢失所有用户与设置）：

```bash
docker compose down -v
docker compose up -d --build
```

---

## 8. 常见问题

### 注册页提交后返回 `{"detail":"Not Found"}`

旧版本（< 1.0.5）需要配置 `APP_REGISTER_PUBLIC_AUTH_URL` 否则前端会请求到 `register_ui` 自身导致 404。请升级到 1.0.5+，或在 `.env` 中正确填写公网可访问的 auth_api URL。

### Turnstile 报错 110200 / 400 Bad Request

site_key 未授权当前 hostname。把当前访问域名（包括 `localhost`）加进 Cloudflare 后台白名单。

### 收不到验证码

- 检查管理后台 SMTP 配置；用"发送测试邮件"
- 检查 `email_log` 表错误信息
- 腾讯企业邮：用户名必须是完整邮箱，密码必须是客户端专用密码

### auth_api 健康检查怎么做

`auth_api` 已不对外暴露端口，可在宿主机上：

```bash
docker compose exec auth_api curl -sf http://127.0.0.1:17306/healthz
```

或通过反代：`curl http://127.0.0.1:3090/api/public/turnstile_site_key`
