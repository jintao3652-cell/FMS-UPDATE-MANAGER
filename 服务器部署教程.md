# FMS 后备隐藏能源登录服务部署教程（1Panel）

本文用于部署你的固定登录接口，供客户端调用：

- `http://data.cnrpg.top:17306/api/auth/login`

当前项目内已提供后端代码目录：

- `deploy/backup_auth`

---

## 1. 准备条件

1. 一台已安装 1Panel 的 Linux 服务器
2. 域名 `data.cnrpg.top` 已解析到服务器公网 IP
3. 服务器开放端口：
- `17306/tcp`（当前直连方式）
- `80/443`（后续 HTTPS 反代推荐）

---

## 2. 上传部署文件

将本地目录上传到服务器，例如：

- 本地：`deploy/backup_auth`
- 服务器：`/opt/fms-backup-auth`

上传后目录应包含：

1. `docker-compose.yml`
2. `Dockerfile`
3. `requirements.txt`
4. `.env.example`
5. `app/`（后端代码）

---

## 3. 配置环境变量

登录服务器，执行：

```bash
cd /opt/fms-backup-auth
cp .env.example .env
```

编辑 `.env`，至少修改以下三个值：

1. `MYSQL_PASSWORD=你的数据库强密码`
2. `MYSQL_ROOT_PASSWORD=你的数据库 root 强密码`
3. `APP_JWT_SECRET=一串足够长的随机密钥`
4. `APP_ADMIN_PASSWORD=你的管理员密码`

可保持默认：

1. `MYSQL_DATABASE=fms_auth`
2. `MYSQL_USER=fms_auth`
3. `APP_ADMIN_USERNAME=admin`
4. `APP_JWT_EXPIRE_MINUTES=60`

---

## 4. 在 1Panel 创建编排应用

1. 打开 1Panel：`容器 -> 编排(Compose) -> 创建`
2. 选择“从目录导入”或“粘贴 YAML”
3. 使用文件：`/opt/fms-backup-auth/docker-compose.yml`
4. 工作目录设置：`/opt/fms-backup-auth`
5. 点击“创建并启动”

启动后应看到两个服务：

1. `db`（PostgreSQL）
2. `auth_api`（FastAPI 登录服务）

---

## 5. 检查服务状态

可在 1Panel 查看，也可命令行检查：

```bash
cd /opt/fms-backup-auth
docker compose ps
docker compose logs -f auth_api
```

健康检查：

```bash
curl http://127.0.0.1:17306/healthz
```

预期返回：

```json
{"ok":true,"service":"fms-backup-auth"}
```

---

## 6. 验证登录接口

执行：

```bash
curl -X POST "http://data.cnrpg.top:17306/api/auth/login" \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"你的管理员密码","client":"FMS UPDATE MANAGER"}'
```

成功示例：

```json
{
  "success": true,
  "message": "ok",
  "token": "xxxxxx",
  "expires_in": 3600,
  "user": {
    "username": "admin",
    "role": "admin"
  }
}
```

---

## 7. 客户端请求格式（已兼容）

客户端发送：

```json
{
  "username": "xxx",
  "password": "xxx",
  "client": "FMS UPDATE MANAGER",
  "timestamp": "2026-..."
}
```

后端返回至少包含：

1. `success`
2. `message`
3. `token`

---

## 8. 常用维护命令

```bash
cd /opt/fms-backup-auth
docker compose up -d --build
docker compose restart auth_api
docker compose logs -f auth_api
docker compose logs -f db
```

数据库备份示例（MySQL）：

```bash
docker exec -t fms-backup-auth-db-1 sh -c 'exec mysqldump -u"$MYSQL_USER" -p"$MYSQL_PASSWORD" "$MYSQL_DATABASE"' > /opt/fms-backup-auth/backup.sql
```

---

## 9. 常见问题排查

1. 无法访问 `:17306`
- 检查安全组、系统防火墙是否放行 17306
- 检查容器是否运行：`docker compose ps`

2. 返回 401
- 账号或密码错误
- 确认 `.env` 的 `APP_ADMIN_PASSWORD` 与测试密码一致

3. 修改 `.env` 后未生效
- 执行：
```bash
docker compose up -d --build
```

4. 域名不通
- 检查 DNS 是否生效，是否指向正确 IP

---

## 10. 生产环境推荐（HTTPS）

建议通过 1Panel 网站反向代理启用 HTTPS：

1. 新建网站，域名填 `data.cnrpg.top`
2. 反代目标填 `127.0.0.1:17306`
3. 申请 Let’s Encrypt 证书
4. 开启 HTTPS 与强制跳转

完成后建议客户端改为：

- `https://data.cnrpg.top/api/auth/login`
